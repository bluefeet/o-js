o.js

Javascript utility belt for rich objects and prototypes. Ooooh!

Why o.js?

JavaScript already has built-in support for objects and inheritance, but it is very lean and only provides the most basic functionality. This library aims to fill in missing the pieces.

Flexible

There are several libraries already that provide rich OO frameworks for JavaScript. This library does not provide a framework, and does not impose standard OO. Use it as you see fit!

Simplicity

One of the tenants of great software design is "do one thing, and do it well". We follow this phylosophy by providing well thought-out bite-sized features which work independently and compliment each other when used together.

An Example

var obj = {
    getName: o.reader('name'),
    setName: o.writer('name'),
    hasName: o.predicate('name'),
    clearName: o.clearer('name'),
    age: o.accessor('_age')
};

obj.setName('Tiberius');
if (obj.hasName()) console.log( obj.getName() );
obj.clearName();

obj.age( 23 );
console.log( obj.age() );

Experimental

o.js is under heavy development and is still quite new. The API will change, and what works now may change later. Please expiriment and provide feedback, enhancements, and bug fixes! If you would you like to contribute then fork our respository on GitHub and send a pull request.

Currently the best avenue of support is by submitting issue reports and feature requests on our GitHub Issue Tracker.

The Core API documentation is available on the o.js Wiki.
