o.js

JavaScript utility belt for rich objects and prototypes. Ooooh!

Why o.js?

JavaScript already has built-in support for objects and inheritance, but it is very lean and only provides the most basic functionality. This library aims to fill in missing the pieces.

Flexible

There are several libraries already that provide rich OO frameworks for JavaScript. This library does not provide a framework, and does not impose standard OO. Use it as you see fit!

Simple

One of the tenants of great software design is "do one thing, and do it well". We follow this phylosophy by providing well thought-out bite-sized features which work independently and compliment each other when used together.

Properties

With o.js you can create rich property definitions such as type, devoid, filter, chain, and more, when using o.reader, o.writer, and o.accessor. Properties can be cleared with o.clearer and checked if the property has been set with o.predicate. Proxy methods, which will call a function on an underlying property, can be created with o.proxy. 

obj.name = o.accessor(
    '_name', {
        type: 'string',
        required: true
    }
);
obj.name('Bilbo');

Constructors

o.construct simplifies the creation of constructor functions by grouping the function and the prototype definitions in one call. o.augment takes this even further by providing a lightweight way to correctly inherit from a parent constructor. 

var Point = o.construct(
    function (x, y) {
        this.x( x );
        this.y( y );
    },
    {
        x: o.accessor('_x'),
        y: o.accessor('_y')
    }
);

Methods

o.before and o.after can be used to add behavior to an object's method without actually changing the interface. With o.around this is taken to the next step by providing the ability to intercept a method's arguments, change them, or even decide not to call the original method. 

obj.setColor = o.writer('_color');

obj.setColor = o.after(
    obj.setColor,
    function () {
        console.log( this._color );
    }
);

obj.setColor( 'blue' );
// Stores 'blue', then logs 'blue'.

Utilities

o.merge merges two or more objects, while o.clone properly clones an object, prototype, constructor, and all. There is also o.has and o.noConflict. 

var obj = o.merge(
    { name:'Bilbo' },
    { age:111 }
);
// obj = { name:'Bilbo', age:111 }

Types

By itself, o.js provides very basic type checking. The optional Types API, o-types.js, extension provides a wide array of types for doing standalone validation, as well as validation of property values before they are written. 

if (o.isNumber(123))
    console.log('ok');

var obj = {
    setName: o.writer(
        '_name',
        { type: o.stringType() }
    )
};
obj.setName( 'Totoro' );

The above is just a taste of what is available. More examples and details are available in the documentation.
